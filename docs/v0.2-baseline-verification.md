## 2. 启动入口

原型目录：

```text

```

启动入口：

```text
frontend/lunatte-v0.2/start-stillgarden.bat
frontend/lunatte-v0.2/start-stillgarden.ps1
frontend/lunatte-v0.2/server.py
```

正常用户启动方式：

```powershell
cd D:\A月亮啊\lunette-v0.2\frontend\lunatte-v0.2
.\start-stillgarden.ps1
```

或双击：

```text
start-stillgarden.bat
```

验证用启动方式：

```powershell
cd D:\A月亮啊\lunette-v0.2\frontend\lunatte-v0.2
$env:STILLGARDEN_PORT = "8877"
$env:STILLGARDEN_NO_OPEN = "1"
python .\server.py
```

说明：

- 默认端口是 `8877`。
- `STILLGARDEN_NO_OPEN=1` 用于自动化验证，避免自动打开浏览器。
- 验证结束后必须关闭服务，避免长期占用端口。

## 6. 页面与功能人工检查清单

这些检查需要在浏览器里完成。每次修改 CSS / JS / `server.py` 后应至少跑一遍。

### 6.1 Home

检查项：

- 页面能打开。
- 手机壳布局没有明显错位。
- 服务灯能显示当前服务状态。
- 统计信息能正常显示。

通过标准：

- 无白屏。
- 无明显重叠。
- 服务在线时显示连接状态。

### 6.2 Chat

检查项：

- 会话列表能打开。
- 林絮、噔噔、Aimas、客厅房间能进入。
- 输入框能输入。
- 不发送真实模型请求时，不应误写错误数据。
- 聊天信息页能打开。
- 请求预览能看到短期 session、长期候选、self 开关、禁止读取列表等信息。

通过标准：

- 房间切换不串 session。
- UI 不遮挡输入框。
- 不误读 V0.1 原始数据。

### 6.3 Moments

检查项：

- 主圈圈流能打开。
- 个人圈圈能打开并返回。
- 动态卡片、头像、评论、回复显示正常。
- 三点菜单能打开。
- 手动请林絮 / 噔噔 / Aimas 回应入口仍存在。
- 自动评论候选入口能打开，但候选预览不应直接写入。

通过标准：

- 列表不白屏。
- 评论框不被刷新打断。
- 自动评论仍需确认才写入。

### 6.4 Archive

检查项：

- Archive 首页连体列表显示。
- 翻抽屉入口可进入。
- “留下这一件事”正式记忆卡可进入。
- 确认记忆列表可刷新。
- “每日回声”“下个月再拆开的信”仍为占位，不误判为已完成。

通过标准：

- 已确认记忆能读取。
- 新记忆必须二次确认后才写入。
- 占位功能不可误触发写入。

### 6.5 Timeline

检查项：

- 底部“时间”页可打开。
- 人物筛选和来源筛选存在。
- 每页 5 条分页存在。
- 新事件在前。
- 卡片默认收起，点击后真实展开并推动后续卡片。

通过标准：

- `/api/timeline` 只读。
- 不迁移、不写入、不向量化。
- 时间排序不明显错乱。

### 6.6 Profile / More

检查项：

- More 页面能打开。
- 我的个人设置能打开。
- 当前状态能打开。
- self 总开关和林絮 / 噔噔 / Aimas 三个子开关能显示。
- 自动评论设置分组能显示。
- Provider 和 Aimas Connector 配置页能打开。

通过标准：

- self 默认关闭时不带入请求。
- self 开关开启后只对允许的住户可读。
- API key 不在前端明文回显。

## 7. 每次改 CSS 后必须复查

重点复查：

- 手机壳外框。
- 底部导航。
- 顶部栏。
- Chat 输入区。
- Moments 卡片和三点菜单。
- Timeline 分页和展开卡片。
- More 当前状态开关。
- Archive 连体列表和二级页。
- 弹层、确认框、toast。

高风险点：

- `overflow: hidden` 裁剪下拉框或菜单。
- 按钮高度不一致。
- 卡片展开只是视觉幻影，没有推动下面内容。
- 文字溢出按钮或卡片。
- 底部导航遮挡内容。

## 8. 每次改 JS 后必须复查

重点复查：

- 页面切换。
- Chat 房间切换。
- 消息发送。
- 请求包生成。
- self 开关状态。
- Moments 评论、回复、菜单、候选确认。
- Archive 写入前二次确认。
- Timeline 筛选、分页、展开。
- 配置保存。

高风险点：

- 事件监听重复绑定。
- 后台刷新打断当前输入。
- localStorage 和 D 盘 JSONL 状态不一致。
- 写入路径指向 V0.1。
- Aimas / provider 路由混用。

## 9. 每次改 server.py 后必须复查

重点复查：

- `/api/stats`
- `/api/config`
- `/api/session`
- `/api/moments`
- `/api/timeline`
- `/api/memory-confirm`
- `/api/moments-auto-preview`
- `/api/aimas-probe`

高风险点：

- JSONL 写入格式破坏。
- `secrets.local.json` 明文泄露。
- 图片资产路径写错。
- CORS/API_BASE_URL 改动影响本地模式。
- 启动端口或服务灯判断异常。

## 10. Baseline Known Issues

以下是当前基线已知问题或未完成功能。后续测试时不要误判成新 bug。

- 历史记录：本清单生成时 `styles.css`、`script.js`、`server.py` 仍是单体文件；截至 2026-06-30，CSS / JS / 后端第一轮拆分已完成，当前状态以 `frontend/lunatte-v0.2/README.md` 和 `docs/v0.2-next-actions.md` 为准。
- Timeline 是只读草稿层，不迁移、不写入、不替代 Archive 确认记忆。
- Moments 自动评论已有候选和确认链路，但还没有后台自动触发器。
- Wake 有收件箱和候选草稿，但还没有真正自主唤醒、通知和调度器。
- Archive 的“每日回声”“下个月再拆开的信”是占位。
- Archive 确认记忆还没有编辑、撤回、删除和权限审计。
- Chat 还没有进房间前预热上下文。
- 跨房间搜索、搜索分页、精细命中高亮还没做完。
- Aimas 能聊天，但“小电脑”工具输出折叠卡和完整 Hermes session 管理还没做。
- 图片缩略图、多图、失败队列、回收站还没做。
- SQLite + FTS5、向量库、RAG 仍未接入。
- Cloudflare/VPS/CORS/远程部署暂缓。
- UI 模板只作为参考，不能直接合并。
- AGY/Gemini 补丁脚本禁止执行。

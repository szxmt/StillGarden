## 5. Archive 确认记忆编辑/撤回/删除/审计方案

为什么先做：

- Archive 已经能写入正式确认记忆，但还缺编辑、撤回、删除和权限审计。
- 这是比大拆 UI 更贴近实际使用的数据安全任务。

涉及文件：

- `frontend/lunatte-v0.2/README.md`
- `sessions/prototype/confirmed-memory.jsonl`
- `sessions/prototype/config.json`
- `docs/lunette-v0.2-plan-reviewed.md`
- 根目录 `memory-vault-usage-v0.1.md`

不允许动哪些文件：

- 不改 `confirmed-memory.jsonl`
- 不改 `script.js`
- 不改 `server.py`
- 不改 V0.1 原始数据

怎么测试：

- 只做方案设计。
- 设计事件追加式数据模型：新增 revision / revoked / deleted 标记，而不是物理删除原记录。
- 设计前端入口和二次确认规则。
- 设计审计显示字段：谁可读、是否 sensitive、何时创建、何时撤回。

成功标准：

- 有可执行的数据模型和 UI 入口方案。
- 明确删除不是直接删文件。
- 明确权限审计如何显示和测试。

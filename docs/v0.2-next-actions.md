### 3. JS 第一轮拆分

为什么做：

- `script.js` 负责状态、渲染、事件、API、数据写入，后续拆分风险高。
- 先把 Web/App 都可能复用的底层逻辑从 DOM 入口里拿出来。

状态：已完成第一轮。

涉及文件：

- `frontend/lunatte-v0.2/script.js`
- `frontend/lunatte-v0.2/shared/lunatte-core.js`

当前结论：

- JS 拆分可以收口。
- `shared/lunatte-core.js` 放通用 token、配置、请求封装、状态读写、格式化和纯数据 helper。
- DOM 渲染、事件绑定、页面切换仍留在 `script.js`，后续按具体功能再拆。

### 4. 后端第一轮拆分

为什么做：

- `server.py` 同时承担配置、路径、存储、routes 和业务逻辑，继续堆功能风险高。
- 本轮优先抽出稳定底层，避免继续无限小刀。

状态：已完成第一轮。

涉及文件：

- `frontend/lunatte-v0.2/server.py`
- `frontend/lunatte-v0.2/server_config.py`
- `frontend/lunatte-v0.2/server_storage.py`

当前结论：

- 后端拆分可以收口。
- `server_config.py` 负责路径、默认配置、常量和 V0.2 数据边界。
- `server_storage.py` 负责 JSON/JSONL、secrets、profile assets、prototype 图片资产和统一 append。
- 复杂 routes、provider 调用、moments/timeline 业务逻辑暂留 `server.py`，等具体功能推动时再拆。

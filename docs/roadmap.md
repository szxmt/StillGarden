# Roadmap

## 当前阶段

阶段：V0.3 总蓝图确认与功能恢复前准备。

结构重构已完成，整体方向以 `全局总蓝图.md` 为准。这里只保留短期执行队列。任何功能开始前先确认写入位置，避免回填到入口文件或最大文件。

## P0：总蓝图收口

- 新建 `全局总蓝图.md`。
- 把 v0.1 / remax / 参考库的有效方向收进总蓝图。
- 保持 README 只做入口，roadmap 只做短期队列。

## P1：UI 方向试点

先研究和定稿，不直接全量替换。

建议试点：

1. Chat 信息页 / 请求预览 UI 化卡片。
2. Moments 卡片密度与评论区。
3. Archive 记忆卡编辑/审计页。

验收：

- token/components 能支撑变化。
- 页面 CSS 不复制主题。
- 不改 API 和数据结构。

## P2：记忆库事件模型

目标：为 SQLite、FTS5、时间线、热力图、RAG 打统一底座。

顺序：

1. 事件 schema 草图。
2. timestamp 标准化规则。
3. JSONL 到 SQLite 副本索引。
4. FTS5 搜索。
5. 热力图聚合。
6. RAG 检索上下文。

## P3：Archive 确认记忆治理

- 编辑。
- 撤回。
- 删除。
- 审计。
- 可读范围变更记录。

必须先设计数据模型和 UI 入口，再写代码。

## P4：自动评论与 Wake

- 自动评论后台触发器。
- 冷却/安静时段执行。
- Wake 主动候选调度。
- 全部写入时间线事件。

## P5：真 App 路线验证

目标不是 WebView 套壳。先验证：

- React Native / Expo 是否能复用 API 协议和设计 token。
- Android APK 侧载流程。
- 本地优先数据和远程同步边界。
- VPS 是否只承担轻量 API / 同步桥。

## 暂缓

- 商店发布。
- Supabase 正式迁移。
- Cloudflare/VPS 正式部署。
- 全量向量库。
- 大规模 UI 重写。

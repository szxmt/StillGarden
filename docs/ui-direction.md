# UI 方向

## 结论

UI 需要现在开始研究，但不应立刻全量替换源码。正确顺序是：定原则、补 token/components、做试点页面、验证后铺开。

## 当前问题

- 原型已经能用，但部分界面仍偏调试工具和网页原型。
- 请求预览、配置页、时间线和 Archive 审计还需要更像正式 App。
- 不能靠每个页面单独重写样式，否则会再次堆出维护问题。

## 视觉方向

参考 Antigravity 文档里的 Abyssal Letter / 私密手账 / 文字 AVG 方向，但不要直接合并模板源码。

关键词：

- 私密。
- 纸感。
- 细线。
- 轻磨砂。
- 信息密度更高。
- 少大卡片。
- 少厚重阴影。
- 小按钮、小状态、小动作。
- 移动端优先。

## 工程规则

- 整体气质：改 `00-tokens.css`。
- 通用控件：改 `05-components.css`。
- 页面布局：改对应页面 CSS。
- 不为新主题复制整套页面 CSS。
- 不直接合并 `docs/archive/imported-antigravity-raw/.../ui模板`。

## UI 试点顺序

1. 请求预览 UI 化卡片。
2. Chat 信息页和 Profile。
3. Moments 卡片密度。
4. Archive 记忆卡编辑/审计。
5. Timeline / 热力图。

每个试点都必须保持 API 和数据结构不变，先只改视觉和布局。

# 小窝交互设想 v0.1

这个文件记录未来客户端的交互方向。

当前原型只是为了验证结构，不代表最终交互。

## 总体方向

最终应该更像一个真正的聊天软件，而不是网页实验页面。

关键词：

```text
底部菜单
独立会话
头像
备注
时间戳
朋友圈入口
单独页面
清爽
```

## 底部菜单

最终形态：

- 菜单固定在底部。
- 类似微信/QQ 那种清爽底栏。
- 不靠长页面上下滚动找功能。
- Home、Chat、Archive、Timeline、More 可以是主入口。
- 有些入口可以合并到同一页里，例如朋友圈/视频号那种二级入口。

当前原型：

- 已经从长页滚动改成底部菜单切页。
- 底部菜单已改成短中文标签。
- 视觉还只是原型，不是最终手机版。

## 会话入口

最终不应该是左上角下拉框选名字。

更适合：

- Chat 首页显示会话列表。
- 点 `林絮` 进入林絮单独会话。
- 点 `噔噔` 进入噔噔单独会话。
- 点 `Aimas 的小窝` 进入 Aimas 单独会话。
- 点 `客厅` 进入群聊/一人多机对话。

每个会话都是独立 session。

不要做：

```text
一个聊天框，左上角切换名字。
```

要做：

```text
会话列表 -> 点进某个会话 -> 进入该 session。
```

## 聊天气泡

最终不需要每条消息头上都标 `我` 或 `林絮`。

原因：

- 聊天软件通过左右气泡、头像、备注和时间戳区分身份。
- 每条都标名字会显得像后台日志，不像聊天。

未来样式：

- 用户消息在右侧。
- 对方消息在左侧。
- 左侧有头像。
- 右侧可以有用户头像或省略。
- 连续消息可以合并头像显示。
- 重要时间点显示时间戳。

## 头像和备注

未来应该支持：

- 每位住户有头像。
- 每位住户有备注名。
- Alice / 林絮 可以显示为用户设置的备注。
- 噔噔可以显示为用户设置的备注。
- Aimas 可以显示为 `Aimas` 或 `Aimas 的小窝`。

头像可以先是：

- 字母头像
- 手绘头像
- 小动物头像
- 用户上传图片

## 时间戳

未来聊天需要正确时间戳。

时间戳规则：

- 新会话开始显示日期。
- 间隔较久显示时间。
- 同一分钟内连续消息可以不重复显示。
- Archive 写入时保留准确时间。

## 朋友圈入口

未来可以有类似朋友圈的入口。

可能形式：

- Home 页里有入口卡。
- More 页里有入口。
- Chat 页顶部有小入口。

交互方式：

- 不直接堆在外面。
- 点进去是单独页面。
- 类似“朋友圈/视频号”：在一个大入口体系里，但内部是自己的页面。

可能内容：

- 日常回声
- 小作品
- 照片/图片
- 日记页
- 时光胶囊
- 主动唤醒记录

## 当前不急着做

- 不急着做最终 App UI。
- 不急着做头像上传。
- 不急着做朋友圈。
- 不急着做真正 EXE。
- 不急着做手机 App。

当前只先把方向存下来。

## 下一步建议

先把当前 Chat 原型从“下拉选择对象”逐渐改成：

```text
Chat 会话列表 -> 点击某个住户 -> 进入该住户单独会话
```

这一步完成后，会更接近真正聊天软件。

当前进展：

- Chat 已有会话列表雏形。
- 林絮、噔噔、Aimas 的小窝已作为独立入口出现。
- 下拉选择对象已从主交互中移除。

# 产品蓝图

## 核心判断

Lunette / 月亮小窝不是模型切换器。它是一个小窝：林絮、噔噔、Aimas 和用户拥有各自房间、关系、时间线、档案和可追溯记忆。

## 人格系统原则

- 不是“聊天人格、评论人格、发圈人格”分开运行。
- 所有入口都回到同一个完整住户系统。
- 旧记录和 RAG 只提供背景、证据、时间线和边界，不替代当下回应。
- 提示词层要薄，不能把表达包成多层任务报告。
- 权限边界要清楚，但不能把住户变成只会执行模板的工具。

## 主要空间

- Home：首页状态和入口。
- Rooms：住户房间。
- Chat：会话和请求预览。
- Archive：确认记忆、搜索、每日回声、时间胶囊。
- Timeline：统一事件线。
- Moments：圈圈动态、评论、互动。
- More：设置、当前状态、provider、Aimas、个人资料。
- Wake：主动候选收件箱。

## 记忆目标

- 聊天记得。
- 圈圈记得。
- 评论记得。
- Archive 确认记忆可审计。
- 时间线可回看。
- 热力图可聚合。
- RAG 可引用，但不复读。

## 长期目标

- Android APK 可安装。
- 真 App，不是网页套壳。
- 本地优先，后续可选同步。
- VPS 可作为轻量 API / 同步桥，不一开始承载全部数据。

# 前端视觉方向 v0.1

这个文件记录小窝前端的视觉方向。

参考图给出的感觉：

- 不是后台管理。
- 不是模型切换器。
- 是粉白、柔软、带纸页和房间感的小窝。
- 功能很复杂，但视觉上像日记、信封、房间、电话、日历和小卡片。

## 关键词

```text
小窝
房间
纸页
口袋
信号
电话
时光胶囊
回声
日记
花园
月亮
```

## 视觉原则

- 粉白为主，但不能糊成一团。
- 低对比、柔光、纸感、细边框。
- 大标题像手账/杂志页。
- 手机卡片像一扇扇房门。
- 数字统计要像纪念物，不像后台指标。
- 路径和技术细节默认藏在高级页。

## 结构原则

不要做：

```text
顶部 Tab：林絮 / 噔噔 / 普通 AI
```

要做：

```text
Home：小窝中央
Rooms：去不同房间
Archive：收起来的东西
Timeline：走过的路
Phone：打电话
More：设置和维护
```

最终形态应该更像微信/QQ 一类的客户端：

- 底部菜单是主入口。
- 不用长页面上下滚动来切换功能。
- 每个一级入口是一页。
- 有些功能是独立页，比如 Chat、Archive、Timeline。
- 有些功能可以集成在同一页，以入口卡片进入，比如朋友圈和视频号那种关系。

## Session 原则

- 林絮房间是单独 session。
- 噔噔房间是单独 session。
- Aimas 的小窝是单独 session。
- 客厅/群聊是单独 session。
- 不要在同一个 session 里只靠名字切换人格。
- 切换房间时，要切换上下文、记忆权限和会话记录。
- shared 可以被带入，但私有记忆不能跨房间自动带入。

## 最终聊天交互

当前下拉选择住户只是原型。

最终应改成：

- Chat 首页是会话列表。
- 点林絮进入林絮 session。
- 点噔噔进入噔噔 session。
- 点 Aimas 的小窝进入 Aimas session。
- 聊天页用头像、备注、左右气泡和时间戳区分身份。
- 不需要每条消息都写 `我` 或 `林絮`。
- 底部菜单固定在底部。
- 朋友圈/视频号式功能可以作为入口进入独立页面。

## 首页应该显示

- 小窝名字
- 两位居民：Alice / 林絮，噔噔
- 未来住户：Aimas
- 小窝天数
- 记忆碎片数
- 日记页数
- 对话片段数
- 最近回声
- 进入房间按钮

## 林絮房间

氛围：

- 月光
- 低声
- 克制
- 安静
- 谈心

元素：

- 林絮名牌
- “低声房间”
- 最近的谈心片段
- 生成上下文草稿
- 读取记忆按钮

## 噔噔房间

氛围：

- 日常
- 活泼
- 工具
- 边界提醒
- 四月前/四月后分相位

元素：

- 噔噔名牌
- 四月前核心 / 四月后边界
- 记忆调用提醒
- 最近活动摘要

## Aimas 的小窝

氛围：

- 终端
- 开源智能体
- 待接入
- 不混线

元素：

- Aimas 门牌
- Hermes 本体说明
- 接口占位
- 不读取旧私有记忆的提醒

## Archive

Archive 像抽屉。

分类：

- Memories
- Diaries
- Snippets
- Bark
- Time Capsules
- Little Works

## Timeline

Timeline 像走过的路。

节点：

- Alice 初始
- 林絮取名
- GPT 导出截止
- 12 月补档
- 噔噔四月前
- 噔噔四月后
- 小窝建立

## 第一版 UI 原型范围

只做静态/本地原型：

- Home
- Rooms
- Archive
- Timeline
- More
- Aimas 终端房间占位
- 小手机 Chat 原型
- 底部菜单多页结构方向
- 不接 API
- 不做登录
- 不做数据库
- 不做主动唤醒

已新增本地桥接方向：

- 静态打开时只看外观。
- 启动本地小窝服务后，按钮可以调用 `memory_context.py`。
- Aimas 暂时只返回接入规则草稿，不读取旧记忆库。

目标：

```text
先让它像一个家，而不是像一堆脚本。
```

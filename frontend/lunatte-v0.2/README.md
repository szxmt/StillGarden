# 月亮小窝 Stillgarden 原型状态板

这是一个本地小窝原型：前端是手机壳 UI，后端是本地 Python 小服务。  
当前快照：2026-06-30

## 进度口径

- 已跑通：可以实际点击、写入或看到结果；可能仍是 JSONL / localStorage，不代表最终架构。
- 雏形：页面和交互方向已确定，但逻辑只做了最小闭环。
- 入口：有按钮或占位，但还没有完整逻辑。
- 未做：没有稳定交互，或必须先做后端 / 数据库 / 调度策略。

## 打开方式

```text
双击 start-stillgarden.bat
```

新版默认使用 `8877`。如果端口被占用，服务会自动换到下一个可用端口并打开页面。  
直接双击 `index.html` 也能看静态外观，但不能写入 D 盘，也不能调用本地记忆工具。

## 重要说明

- 第一条消息也会进入上下文：现在是“发送后先写入当前房间，再按这条消息和最近记录生成请求包”。  
- 还没有“进房间前预热上下文”：进入房间时不会提前生成一包可用记忆，也不会提前检索长期记忆。
- 新聊天会进入短期 session，上下文能接住最近对话；但不会自动成为长期记忆。
- 想让事情明天也稳定记得，需要到 Archive 手写或生成草稿，再确认入库。
- Aimas 当前走独立 Agent Connector，不走 provider 下拉；对 Aimas 来说这里主要是一个聊天入口，它能看到的是实际发过去的消息和它自己那边的能力。
- “我”的圈圈昵称保存在 `config.json` 的 `user_profile.nickname`，圈圈读取和新写入都会用这个昵称；旧事件原文不物理改写。
- 圈圈自动评论设置保存在 `config.json` 的 `moments_auto_comments`，只表示允许范围和节奏；候选预览会检查安静时段、冷却和是否已回应同一条动态；候选确认会复用现有真实评论写入路径。
- self 默认不可读、不可看、不会自动带入任何模型上下文；只有在 More 当前状态里打开 self 总开关并允许对应住户可读时，请求包才会带入 self。
- 图片口径：头像、个人背景、聊天背景、圈圈封面、圈圈正文图片都已接入 D 盘资产库；缩略图、多图和回收站还没做。
- 服务口径：只有 `8877` 上的小窝 Python 服务真的响应时才算“已连接”；如果浏览器里还是绿灯但黑框已关，优先刷新页面或重新双击 bat，因为可能看的是旧页面状态。
- 模型口径：现在的 API 回复仍依赖 provider 自身表现；复读护栏只拦“明显复用旧回复”，不是完整人格/风格控制器。
- 人格 / RAG 口径：林絮、噔噔和 Aimas 不能被做成“旧记录模仿器”或只会套模板的木偶。后续向量库 / RAG 只提供可用背景、证据、时间线和边界，不替代她们当下生成回应；提示词层要尽量薄，权限边界要清楚但不能把表达包成多层任务报告。
- 整体人格口径：最终目标不是“聊天人格、评论人格、发圈人格”分开运行，而是所有入口都回到同一个完整住户系统：由用户发出请求，住户系统接收、结合自身记忆/时间线/当前场景处理，再由同一个住户回应。圈圈评论、聊天回复、自动发圈、主动唤醒都应该进入同一条可追溯时间线。
- 统一时间线口径：当前 `/api/timeline` 和“时间”页只是后续数据库和向量库的只读草稿层，用来先确认“哪些东西应该被记得”；它不会改变现有 JSONL，也不会替代 Archive 确认记忆。

## 隔离规则

- 林絮：默认读取 GPT / Alice / 林絮线和 shared，不读取噔噔私有记忆。
- 噔噔：默认读取 Gemini / 噔噔线和 shared，不读取林絮私有记忆。

- self：默认关闭；More 当前状态里有总开关和“林絮可读 / 噔噔可读 / Aimas 可读”三个子开关。
